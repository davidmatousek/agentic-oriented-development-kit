# Product Documentation

**Location**: `docs/product/`
**Purpose**: Strategic product artifacts — vision, PRDs, roadmap, customer journeys, user stories, OKRs, and the live backlog.

This is the canonical home for product-led work in the AOD lifecycle. PMs draft and maintain these documents; specs and plans under `specs/` reference them.

## Product Index

| Subdirectory | Purpose |
|--------------|---------|
| [01_Product_Vision/](01_Product_Vision/README.md) | Mission, vision, target users, competitive landscape |
| [02_PRD/](02_PRD/INDEX.md) | Product Requirements Documents (one per feature) |
| [03_Product_Roadmap/](03_Product_Roadmap/README.md) | Phase plans and quarterly roadmap snapshots |
| [04_Journey_Maps/](04_Journey_Maps/README.md) | Customer journey maps and touchpoint analyses |
| [05_User_Stories/](05_User_Stories/README.md) | Validated user stories from the discovery backlog |
| [06_OKRs/](06_OKRs/README.md) | Quarterly Objectives & Key Results |
| [_backlog/](_backlog/BACKLOG.md) | Live backlog auto-regenerated from GitHub Issues |

## Lifecycle Touchpoints

Product docs in this directory are read by `/aod.define` (PRD authoring), `/aod.discover` (idea capture and ICE scoring), and the Triad governance gates throughout the AOD lifecycle. Each subdirectory has its own README explaining its role and conventions.
